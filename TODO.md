## 0. 테스트 계정 생성

Supabase 인증이 이메일 기반이고 비밀번호가 8자 이상이어야 하므로 다음 계정을 사용합니다.

### 일반 사용자 1

- 닉네임: `user01`
- 이메일: `user01@test.com`
- 비밀번호: `user01user01`

### 일반 사용자 2

- 닉네임: `user02`
- 이메일: `user02@test.com`
- 비밀번호: `user02user02`

### 관리자

- 닉네임: `admin`

- 이메일: `admin@test.com`

- 비밀번호: `adminadmin`

- [ ] `user01@test.com` 회원가입

- [ ] `user02@test.com` 회원가입

- [ ] `admin@test.com` 회원가입

- [ ] Supabase Authentication의 Users에서 3개 계정 생성 확인

- [ ] Supabase Table Editor의 `profiles`에서 3개 프로필 자동 생성 확인

- [ ] `user01`, `user02`의 `role`이 `citizen`인지 확인

- [ ] `admin` 계정의 `role`을 `admin`으로 변경

관리자 권한 변경 SQL:

```sql
-- 테스트 관리자 계정 지정
update public.profiles
set role = 'admin'
where id = (
  select id
  from auth.users
  where email = 'admin@test.com'
);
```

각 기능은 다음 중 하나로 표시합니다.

```text
✅ 정상
⚠️ 일부 문제
❌ 실행 불가
```

오류 보고 예시:

```text
기능: 교통일지 수정
계정: user01@test.com
URL: http://localhost:3000/journal
결과: ❌ 실행 불가
현상: 수정 내용 저장 클릭 후 기존 내용으로 돌아감
예상: 수정한 출발지와 이동 시간이 목록에 반영
오류 메시지: 콘솔 메시지 전체 첨부
화면 캡처: 첨부
```
