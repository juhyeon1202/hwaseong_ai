# Handoff: 화성특례시 시민 참여형 대중교통 일지 플랫폼

## Overview
화성특례시 시민이 대중교통(시내버스·마을버스) 이용 경험을 기록·공유하고, 지역별 참여율·만족도를 지도로 시각화하며, AI가 민원보고서 대리 작성과 지연 감지 시 똑버스(DRT) 자동 호출을 수행하는 시민 참여형 플랫폼. 시민용 앱(반응형 웹)과 관리자(시청) 대시보드로 구성된다.

## About the Design Files
이 번들의 파일(`교통일지 와이어프레임.dc.html`)은 **HTML로 만든 저해상도 와이어프레임(디자인 레퍼런스)** 이다. 프로덕션 코드가 아니며 그대로 복사해 배포하지 않는다. 목표는 이 화면 구조·흐름·문구를 **실제 개발 환경에서 재구현**하는 것이다.
- 기존 코드베이스가 없다면 아래 "권장 스택"으로 새 프로젝트를 구성해 구현한다.
- 스타일(색/타이포/간격)은 저해상도 와이어프레임을 그대로 픽셀 복제하지 말고, 아래 디자인 토큰과 공공기관 톤(Pretendard, 주황 포인트)을 기준으로 실제 디자인 시스템을 적용한다.

## Fidelity
**Low-fidelity (lofi) 와이어프레임.** 레이아웃·정보구조·플로우·문구의 가이드로 사용한다. 색/타이포/간격은 디자인 토큰을 출발점으로 삼되 최종 UI 디테일은 구현 단계에서 확정한다.

## 권장 스택
- 프론트엔드: React + TypeScript (Next.js 권장), 반응형
- 지도: **카카오맵 JS SDK** (와이어프레임의 OpenStreetMap/Leaflet은 자리표시이며 카카오맵으로 교체). 마커=원형(참여율/만족도 색·크기), 폴리라인=희망 노선/경로, 폴리곤=행정동 경계(선택)
- 인증: 카카오 소셜 로그인 + PASS 본인인증(전화 OTP)
- 백엔드: 회원·포인트·게시판·신고·일지·희망노선 API + DB
- AI 서비스(별도): 민원보고서 생성, 지연 감지·똑버스 자동 호출, 민원 자동 분류/요약, 답변 초안
- 서체: Pretendard

## 브레이크포인트 / 반응형 규칙 (모든 화면 필수)
- 모바일 ~600px / 태블릿 600–1024px / 데스크톱 1024px+
- 네비게이션: 모바일 하단 탭바 → 데스크톱 상단 내비바
- 화면 유형별 재배치:
  - **폼형**(로그인·가입·작성·수정): 모바일 전체폭 1열 → 데스크톱 가운데 정렬 카드(최대 ~480px)
  - **리스트·피드형**(게시판·기록·당첨): 1열 → 2열 → 3열 카드 그리드
  - **지도형**(홈·길찾기·희망노선): 모바일 지도 위/패널 아래 → 데스크톱 지도(좌)+패널(우) 분할
  - **상세형**(게시물·노선·일지): 모바일 1열 세로 → 데스크톱 본문(좌)+요약·액션(우) 2열
  - **관리자형**(대시보드·표): 데스크톱 사이드바+넓은 표 → 모바일 사이드바=상단 메뉴, 표=카드 리스트
- 터치 타깃 ≥ 44px, 지도·이미지 컨테이너는 유동 폭

## Screens / Views

### A. 공통 / 온보딩
1. **로그인** — 카카오로 시작하기(옐로 #FEE500 버튼), 전화번호 로그인, 다국어(한/EN/中/VI). 로고 락업(화성특례시 CI 자리).
2. **간편가입** — 카카오/PASS 본인인증으로 정보 입력 최소화. 닉네임 자동 생성, 본인인증 시 연령/성별 자동, 거주지역은 가입 후 설정.
3. **회원가입(단계형)** — ①약관 동의(전체/필수/선택) ②정보 입력(랜덤 닉네임 자동생성+다시생성, 아이디, 전화+OTP 인증, 비밀번호/확인, 생년월일, 성별[남/여, 다른 사용자 비공개·통계용], 거주 읍면동, 언어, 추천인 코드[선택, 입력 시 +300P]) ③완료(가입 축하 지급).

### B. 시민용 핵심
4. **메인 홈** — 상단: 실제 지도 위 참여율 온도계(행정동 원형 마커, 진할수록 참여 활발). "우리 동네 순위" 경쟁 배너("○○동 N위 · 2위까지 X건"), 빠른 실행(기록/길찾기/룰렛), 실시간 동네 순위 막대. 메인에 참여율+만족도 함께 노출.
5. **동네 참여 랭킹(전체)** — 포디움(1·2·3위) + 전체 행정동 순위 + 우리 동네 독려.
6. **지도 홈** — 정류장·노선 검색, 지도 마커, "＋ 교통일지 기록" FAB.
7. **길찾기** — ①출발·도착 입력(집/회사·즐겨찾는 정류장 불러오기, 최근 검색) ②경로 비교(추천/최소시간/최소환승/최소도보; 각 경로는 **한 줄 색구분 막대**로 구간 표시: 도보=회색, 버스=노선별 색, 폭=소요시간; 라벨 "도보 5분 · 100번 14분 · 도보 3분") ③단계별 실시간 안내(지도+도보→승차→하차 타임라인).
8. **희망 노선 만들기** — ①정류장 검색(이름·번호, **같은 이름의 다른 정류장 존재 시 "정류장 번호 확인" 경고 문구**, 결과에 정류장 번호 배지) ②지도에 핀 순서대로 찍어 선 연결(되돌리기) ③순서 확정(드래그 정렬)·이름·사유·제출.
9. **희망 노선 확장** — 다른 시민 노선 공감·투표(공감순/우리 동네), AI 경로 자동 추천(출발·도착만 찍으면 중간 정류장 제안), 노선 상세·공유(정류장 목록+번호, 공감, 카카오/링크 공유), 관리자 수요 히트맵(지도 위 폴리라인 굵기=투표수).
10. **원터치 익명 신고(정류장)** — 로그인 필요, 신고 데이터엔 개인식별자 미저장. 3색 원터치 버튼: 빨강=만차 통과, 노랑=배차 지연, 초록=환승 실패. 정류장 GPS 자동. 저장 데이터=[정류장ID]/[시간]/[불편유형]. 접수 완료 화면 + 정류장 실시간 익명 집계(유형별 카운트 막대).

### C. 오늘의 일지 (경로 기록형)
11. **카테고리 원클릭 기록** — 출근/퇴근/등하교/기타 버튼만 누르면 날짜·시간 자동 입력.
12. **추천 경로 선택 → 구간 생성** — 카카오맵 추천 경로 비교(색구분 막대) 중 선택 → 구간별 만족도 버튼 자동 생성.
13. **구간별 만족/불만족 기록** — 각 구간(도보 N분/버스 N분)마다 만족/불만족 버튼, 불만족 시 원인(배차지연·혼잡·무정차·불친절·시설) 복수 선택, 메모.
14. **오늘의 일지** — 하루 이동 타임라인(출근/퇴근), 이용시간·응모권 요약.
15. **월별 모아보기** — 이용량 히트맵 캘린더, 누적 통계, 자주 탄 노선.

### D. 커뮤니티 게시판 (익명)
16. **게시판 목록** — 필터: 카테고리(전체/노선 신청/노선 건의/정보공유/질문) + 버스 종류(전체/마을버스/시내버스) + 정렬(최신순/조회순/인기순). 각 글 카드에 카테고리·버스종류 태그, 조회/공감/댓글(노선 신청 글은 투표수).
17. **게시물 작성** — 익명 등록(닉네임/이름 미표시). 카테고리, 버스 종류(마을/시내), 제목, 내용. **버스 이동 동선: 최소 5개 정류장 필수**(정류장 이름·번호 검색, **겹치는 정류장 경고 문구**, 각 정류장 번호 배지, 순서 정렬).
18. **게시물 상세** — 본문, 노선 신청 글은 **투표 진행바+투표하기**, 공유(카카오/링크). **비밀댓글**: 게시물 작성자와 댓글 작성자에게만 공개(그 외 열람 불가).
19. **게시물 신고** — 신고 대상 미리보기, 사유(스팸·광고/욕설·비방/허위정보/개인정보 노출/기타), 상세, 익명 처리.

### E. 이벤트 / 포인트
20. **룰렛** — 원형 룰렛 + "돌리기(응모권 사용)", 보유 응모권 표시. (하단 상품 칩 없음)
21. **출석 체크** — 7일 스탬프(하루 +5P, 7일 연속 보너스 +10P), 연속 일수.
22. **포인트 적립 규칙** — 출석 +5P(1일 1회), 7일 연속 +10P, 게시글 작성 +10P(1일 1회), 일지 기록 적립. 부정 적립 방지 1일 한도.
23. **뽑기권 전환** — **룰렛 뽑기권 1회 = 300P**. 수량 스텝퍼, 잔여 표시.
24. **추천인(친구 초대)** — 내 추천 코드, 카카오/링크 공유, 초대 현황. 추천인 +500P / 신규 가입자 +300P 양쪽 지급. 신규 코드 입력 화면, 포인트 적립·사용 내역.
25. **당첨 내역** — 미사용 상품(바코드/쿠폰, 사용완료 처리), 지난 당첨/미당첨 내역.

### F. 마이페이지 / 계정
26. **마이 홈** — 아바타(프로필 사진 없음, 아바타로 대체)+닉네임+거주동, 통계(내 기록/응모권/당첨). 즐겨찾기(자주 가는 장소=집/회사 고정, 자주 가는 동선, 저장한 장소·정류장). 메뉴: 내 기록 내역, 응모·당첨 내역, 포인트·교환 내역, 문의하기, 알림 설정.
27. **내 기록 내역** — 필터(전체/처리중/완료), 기록 카드(제목·날짜·상태). (사진 없음)
28. **설정** — 프로필 편집, 언어, 알림, 개인정보·약관, 고객센터·문의, 로그아웃.
29. **프로필 편집** — 아바타 변경, 닉네임 다시 생성, 거주 지역, 언어. 생년월일·성별은 비공개·변경 불가. 회원 탈퇴.
30. **회원정보(계정) 수정** — 프로필(아바타/닉네임/지역/언어) + 계정(아이디[변경 불가], 전화번호 변경·재인증, 이메일, 비밀번호 변경). 알림 설정.

### G. 관리자 (시청) 대시보드
31. **지도 대시보드** — 사이드바 내비. 누적/오늘/참여 통계, 지도 히트맵, 유형별 통계 차트.
32. **기록 관리** — 필터(전체/대기/처리), 표(번호·정류장/노선·유형·작성자·상태), 상세·처리 완료.
33. **AI 자동 분류(트리아지)** — 자동 분류·중복 병합·우선순위 큐, 일괄 접수.
34. **AI 인사이트 리포트** — 주간 요약, 이상 감지(급증 감지), 유형별 추이, 참여자 인구통계(성별·연령, 관리자 전용).
35. **AI 답변 초안** — 민원별 답변 자동 생성(정중체/사과/영어 옵션), 발송.
36. **AI 똑버스 관제** — 기준 구역 반경 2.5km 지연 감지 현황 표(자동 호출/승인 대기/모니터링), 지도, 대응 대기 인원, 호출 승인.
37. **AI 자율 대응** — 상황 자동 인식(예: "○○ 정류장 10분간 N명 미탑승") → [시민] 우회 알림 자율 발송("200m 옆 C정류장에서 타 노선 탑승 가능") + [시청] 똑버스 긴급 배치 권고 공문 자동 생성·시그널 전송. 자율 대응 타임라인(관리자 개입/중단 가능).
38. **지역 참여율** — 행정동 온도계 히트맵(지도)+범례+순위(저조 지역=홍보 대상).
39. **만족도 대시보드** — 행정동/노선 만족도 온도계(빨강↔초록), 순위, 불만족 원인.
40. **교통 취약지/사각지대** — 행정동 단위 취약 지도(빨강=취약)+표(야간 공백·미탑승·배차간격·취약 등급), 개선안 생성.
41. **회원 관리** — 회원 통계·검색·필터(정상/정지/신고), 표(닉네임·가입일·연령/성별·기록·상태), 상세·경고·정지.

## Interactions & Behavior
- 카카오 로그인 → 최초면 약관 동의 → 홈. PASS 본인인증 시 연령/성별 자동 취득.
- 지도: 마커 탭 → 정류장/동 상세. 히트맵 마커 색·크기는 지표값 기준.
- 길찾기: 경로 선택 → 단계별 안내 → (선택) 오늘의 일지로 저장.
- 일지: 카테고리 원클릭 → 추천 경로 선택 → 구간 버튼 자동 생성 → 만족/불만족 입력 → 저장. 데이터는 관리자 만족도·마이페이지로 전달.
- 원터치 신고: 버튼 1탭 즉시 접수(익명), 실시간 집계 반영, 임계 초과 시 AI 똑버스 검토.
- 투표/공감: 낙관적 업데이트, 중복 방지.

## Assets
- **화성특례시 CI 로고**: 와이어프레임엔 자리표시(“CI” + 워드마크)만 있음. 실제 공식 CI 이미지로 교체 필요.
